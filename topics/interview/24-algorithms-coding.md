## Краткий разбор тем

<a id="q1"></a>
### Big-O
**Краткий ответ:** Асимптотика времени/памяти от размера входа. Частые классы: O(1), O(log n), O(n), O(n log n), O(n²), O(2ⁿ). Уметь оценивать сложность своего решения и память. Амортизированная сложность (например, `List.Add`).

<a id="q2"></a>
### Two pointers, sliding window, префиксные суммы
**Краткий ответ:**
- **Two pointers** — два индекса навстречу/вдогонку (отсортированный массив, пара с суммой).
- **Sliding window** — подотрезок переменной длины (максимум суммы окна, подстрока без повторов) за O(n).
- **Prefix sums** — предподсчёт сумм для запросов диапазона за O(1).

<a id="q3"></a>
### Хеш-таблицы
**Краткий ответ:** Главный приём «из O(n²) в O(n)»: хранить виденные элементы/счётчики в `Dictionary`/`HashSet` для O(1) поиска (two sum, дубликаты, частоты).

<a id="q4"></a>
### Стек/очередь
**Краткий ответ:** Стек — для парных структур (скобки, монотонный стек, обход без рекурсии). Очередь — BFS, скользящий максимум (deque).

<a id="q5"></a>
### Связные списки
**Краткий ответ:** Разворот списка, поиск цикла (быстрый/медленный указатель — Флойд), нахождение середины, слияние отсортированных.

<a id="q6"></a>
### Деревья
**Краткий ответ:** Обходы DFS (pre/in/post-order) и BFS (по уровням). BST: in-order даёт отсортированную последовательность; поиск/вставка O(log n) в сбалансированном. Высота, LCA, валидация BST.

<a id="q7"></a>
### Графы
**Краткий ответ:** Представление (список смежности). BFS (кратчайший путь в невзвешенном), DFS (компоненты, циклы), топологическая сортировка (DAG), Дейкстра (взвешенный без отрицательных рёбер), Union-Find.

<a id="q8"></a>
### Сортировки
**Краткий ответ:** QuickSort (средн. O(n log n), in-place, нестабильна), MergeSort (O(n log n), стабильна, доп. память), HeapSort. В .NET `Array.Sort` — introsort; `OrderBy` — стабильна. Знать, когда нужна стабильность.

<a id="q9"></a>
### Бинарный поиск
**Краткий ответ:** O(log n) на отсортированном. Вариации: первый/последний элемент ≥ x, поиск по ответу (binary search on answer). Осторожно с границами (off-by-one) и переполнением `(lo+hi)/2`.

<a id="q10"></a>
### Рекурсия/мемоизация
**Краткий ответ:** Рекурсия + кэш результатов подзадач (top-down DP). Следить за глубиной (риск StackOverflow), переводить в итерацию при необходимости.

<a id="q11"></a>
### Динамическое программирование
**Краткий ответ:** Разбиение на перекрывающиеся подзадачи с оптимальной подструктурой. Top-down (мемоизация) или bottom-up (таблица). Классика: рюкзак, LIS, LCS, монеты, пути в сетке.

<a id="q12"></a>
### Жадные алгоритмы
**Краткий ответ:** Локально оптимальный выбор → глобальный оптимум (если задача обладает greedy-свойством). Примеры: интервалы (по концу), сдача монетами (для канонических систем). Требует доказательства корректности.

<a id="q13"></a>
### Битовые операции
**Краткий ответ:** AND/OR/XOR/сдвиги. Приёмы: XOR для поиска уникального, `n & (n-1)` сбрасывает младший бит, проверка степени двойки, битовые маски для подмножеств.

## Подход к live coding
1. Уточнить условие и краевые случаи (пустой вход, дубликаты, переполнение).
2. Проговорить наивное решение и его сложность.
3. Оптимизировать, объясняя ход мысли.
4. Написать чистый код, проверить на примерах вслух.
5. Обсудить сложность и возможные улучшения.

## Решённые задачи

<a id="q14"></a>
### Задача: Two Sum
**Условие:** найти индексы двух чисел с заданной суммой.
**Подход:** хеш-таблица «значение → индекс», один проход.
```csharp
int[] TwoSum(int[] nums, int target)
{
    var seen = new Dictionary<int, int>();
    for (int i = 0; i < nums.Length; i++)
    {
        if (seen.TryGetValue(target - nums[i], out int j)) return [j, i];
        seen[nums[i]] = i;
    }
    return [];
}
```
**Сложность:** время O(n), память O(n).
